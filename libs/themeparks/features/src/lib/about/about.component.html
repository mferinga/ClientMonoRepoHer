Casus omschrijving

De belangrijkste functie van de website is voor dat een gebruiker pretpark kan vergelijken. Een user moet tickets kunnen kopen op de website en parken met elkaar kunnen vergelijken.

Het vergelijken van parken kan op basis van verschillende criterea gaan gebeuren. Ten eerste worden de parken vergeleken op doelgroep zo krijg je geen familiepark aangeraden, wanneer je op zoek bent naar een park met heftige attracties. Ook kan de site parken aangeraden die in de buurt liggen van je locatie, de straal waarin de website van je locatie zoekt kan je Ook zelf aan passen. Als laatste kan de site attractiesmet elkaar vergelijken en op basis daarvan een aanrading doen voor een pretpark.

Tot zover de attractieparken voor de particulier een andere functionaliteit van de website is het sollicteren voor een baan bij een attractiepark. Je kan dan een lijst opvragen van 'JobOffers', deze joboffers zijn voor attracties in het attractiepark. Deze functie kan handig zijn voor mensen die zoeken naar een zeer tijdelijk baan binnen een park. Denk hierbij aan acteurs voor bijvoorbeeld halloween (scare-actor) of kerst en oud en nieuw (waar veel extra entertainment is).

Als laatste kan je als employee de attractie waar je werkt aanpassen. Ook kan je een attractie toevoegen en verwijderen. JobOffers kunnen ook door employees worden aangemaakt, aangepast en verwijderd, dit kan alleen voor een attractie waar die employee voor werkt. Een "normale" user kan hier dan vervolgens op reageren.

korte samenvatting

Een employee is een user met meer functionaliteiten. Beide hebben CRUD functionaliteiten voor het aanmaken van een user (zichzelf). Ook kunnen ze zichzelf aanpassen, bekijken en verwijderen. Een employee mag ook attracties aanmaken, hij/zij mag ook de attractie waar die werkt aanpassen. Zowel de user als de employee mogen de attractie bekijken. Een themepark kan niet doormiddel van CRUD worden aangemaakt of geupdate, dit moet gebeuren in de database zelf. Wel kunnen users en employees parken bekijken. Als laatste heeft een employee toegang tot de CRUD van een attractie. Een employee kan een sollicitatie opstellen voor de attractie waar die werkt, deze kan die vervolgens ook updaten en verwijderen. Zowel de user als de employee kan deze bekijken.
